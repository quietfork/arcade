# Raster icon set

Square app/social icons in nine sizes, generated by downscaling [`arcade/build/appicon.png`](../../arcade/build/appicon.png) (the actual in-app icon, 1254×1254 master).

| File | Size | Typical use |
|---|---|---|
| `arcade-16.png` | 16×16 | Browser favicon (legacy) |
| `arcade-32.png` | 32×32 | Browser favicon (standard) |
| `arcade-64.png` | 64×64 | Browser favicon (retina) |
| `arcade-180.png` | 180×180 | `apple-touch-icon` (iOS home screen) |
| `arcade-192.png` | 192×192 | Android / PWA manifest |
| `arcade-240.png` | 240×240 | **Product Hunt avatar** |
| `arcade-256.png` | 256×256 | General use, dev.to cover |
| `arcade-512.png` | 512×512 | High-res social, Discord server icon |
| `arcade-1024.png` | 1024×1024 | App Store / hi-res master |

All generated via PowerShell + `System.Drawing` with high-quality bicubic interpolation.

## When to regenerate

If the brand icon ever changes, update `arcade/build/appicon.png` (the source of truth — also used by Wails to build the app icon), then re-run the resize script. Keep this set in sync with the in-app icon.

## Regen command (PowerShell)

```powershell
Add-Type -AssemblyName System.Drawing
$src = 'arcade/build/appicon.png'
$outDir = 'assets/icon'
foreach ($s in 16,32,64,180,192,240,256,512,1024) {
  $img = [System.Drawing.Image]::FromFile($src)
  $bmp = New-Object System.Drawing.Bitmap $s, $s
  $g = [System.Drawing.Graphics]::FromImage($bmp)
  $g.InterpolationMode = 'HighQualityBicubic'
  $g.DrawImage($img, 0, 0, $s, $s)
  $bmp.Save("$outDir/arcade-$s.png", 'Png')
  $g.Dispose(); $bmp.Dispose(); $img.Dispose()
}
```

For favicon `.ico` bundling (combine 16+32+64 into one file), use ImageMagick:

```bash
magick assets/icon/arcade-16.png assets/icon/arcade-32.png assets/icon/arcade-64.png assets/icon/favicon.ico
```
